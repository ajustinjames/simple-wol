# Issue #11 — Fix SanitizeName double-encoding display bug

**Status:** refined
**Branch:** —
**Issue:** https://github.com/ajustinjames/simple-wol/issues/11

## Spec

### Problem

`SanitizeName()` in `devices.go:57-64` performs HTML entity encoding (`<` → `&lt;`, `>` → `&gt;`, `"` → `&quot;`, `'` → `&#39;`) before storing in the database. The frontend renders device names via `nameTd.textContent = d.name` (`app.js:80`), which auto-escapes HTML entities. This double-encoding means `My <PC>` is displayed as the literal text `My &lt;PC&gt;`.

### Implementation

**Simplify `SanitizeName` in `devices.go`** to only trim whitespace:
```go
func SanitizeName(name string) string {
    return strings.TrimSpace(name)
}
```

The HTML entity encoding is unnecessary because:
- The frontend uses `textContent` (not `innerHTML`), which is XSS-safe
- The API returns JSON, which is consumed by JavaScript — HTML encoding in JSON is wrong
- Data should be stored in its raw form; encoding is a presentation concern

**Files:** `devices.go`

### Test Cases

- [x] Device name with `<>` characters is stored without encoding
- [x] Device name with `"` and `'` characters is stored without encoding
- [x] Whitespace-only name is trimmed to empty string
- [x] Name with leading/trailing whitespace is trimmed
- [x] Existing `SanitizeName` tests updated to reflect new behavior

## Decisions

1. **Remove encoding entirely rather than move it to render time** — There is no server-side HTML rendering of device names. The Go templates serve the page shell; device names come from the JSON API and are rendered by JavaScript using `textContent`. There is no render-time location on the server to add encoding.

2. **No character allowlist** — Device names like "My <PC>" or "Dad's Desktop" are valid. Restricting characters would surprise users. The frontend is already safe against XSS via `textContent`.

3. **No database migration for existing data** — Existing data with encoded entities will display incorrectly (`&lt;` shown literally). However, this is a new app with few users, and re-saving a device via the edit flow will fix it. A migration adds complexity for a near-zero-impact scenario.
