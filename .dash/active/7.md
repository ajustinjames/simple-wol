# Issue #7 — Clean up expired sessions from database

**Status:** refined
**Branch:** —
**Issue:** https://github.com/ajustinjames/simple-wol/issues/7

## Spec

### Problem

Sessions are validated against `expires_at` but expired rows are never deleted from the `sessions` table. Over time this causes unbounded database growth.

### Implementation

**Add a `CleanExpiredSessions` function in `auth.go`:**
```go
func CleanExpiredSessions(db *sql.DB) (int64, error) {
    result, err := db.Exec("DELETE FROM sessions WHERE expires_at < ?", time.Now())
    if err != nil { return 0, err }
    return result.RowsAffected()
}
```

**Add a `startSessionCleanup` method on `App` in `main.go`** that launches a goroutine running `CleanExpiredSessions` on a ticker (every 1 hour). Log the count of deleted sessions at debug/info level.

**Call `app.startSessionCleanup()` in `main()` after creating the `App` struct** (around main.go:57).

**Files:** `auth.go`, `main.go`

### Test Cases

- [x] `CleanExpiredSessions` deletes sessions with `expires_at` in the past
- [x] `CleanExpiredSessions` does not delete sessions with `expires_at` in the future
- [x] Returns correct count of deleted rows
- [x] No error when no expired sessions exist

## Decisions

1. **Background goroutine over cleanup-on-validation** — Cleaning during validation adds latency to every authenticated request. A background goroutine keeps the hot path fast and is the standard pattern for periodic maintenance.

2. **1-hour interval** — Sessions last 24 hours minimum. Hourly cleanup is frequent enough to prevent meaningful buildup without being wasteful.

3. **No graceful shutdown of the cleanup goroutine** — The app has no graceful shutdown mechanism today. The goroutine will simply exit when the process dies. This is fine for a simple home tool.
