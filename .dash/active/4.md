# Issue #4 — Add HTTP security headers middleware

**Status:** refined
**Branch:** —
**Issue:** https://github.com/ajustinjames/simple-wol/issues/4

## Spec

### Problem

No HTTP security headers are set on any response from the server. This is the highest-severity finding in the audit. The application serves HTML pages and a JSON API with zero browser security directives.

### Implementation

**Add a `securityHeaders` middleware function in `main.go`** that wraps the root `mux` and sets the following headers on every response:

| Header | Value | Notes |
|--------|-------|-------|
| `Content-Security-Policy` | `default-src 'self'; script-src 'self'; style-src 'self'; img-src 'self'; connect-src 'self'` | Disallows inline scripts/styles |
| `X-Content-Type-Options` | `nosniff` | |
| `X-Frame-Options` | `DENY` | |
| `Strict-Transport-Security` | `max-age=31536000; includeSubDomains` | Only set when `isSecure(r)` is true |
| `Referrer-Policy` | `strict-origin-when-cross-origin` | |
| `Permissions-Policy` | `camera=(), microphone=(), geolocation=()` | Belt-and-suspenders |

**Move inline scripts from `login.html` to a separate `static/login.js` file.** Currently `login.html:52-116` contains `<script>` with `handleSetup`, `handleLogin`, and the setup-success query param check. Move this to `static/login.js` and reference it with `<script src="/static/login.js"></script>`. This allows the strict CSP to work without `unsafe-inline`.

**Wire the middleware in `main.go`** by wrapping `mux` before passing to `http.ListenAndServe`:
```go
if err := http.ListenAndServe(":"+port, securityHeaders(mux)); err != nil {
```

**Files:** `main.go`, `static/login.html`, `static/login.js` (new)

### Test Cases

- [ ] All responses include `Content-Security-Policy` header
- [ ] All responses include `X-Content-Type-Options: nosniff`
- [ ] All responses include `X-Frame-Options: DENY`
- [ ] HSTS header is set when request has `X-Forwarded-Proto: https`
- [ ] HSTS header is NOT set on plain HTTP requests
- [ ] All responses include `Referrer-Policy`
- [ ] Login page loads and functions correctly with external script (no inline JS)
- [ ] Setup flow still works after moving script to `login.js`

## Decisions

1. **Middleware wrapping the mux vs. per-handler headers** — A single middleware is cleaner, ensures no endpoint is missed, and is the standard Go pattern.

2. **Move inline JS to separate file rather than use CSP nonces** — Nonces require server-side template injection of a random value per request. A separate static file is simpler and works with `embed.FS` without changes.

3. **HSTS conditional on secure context** — Setting HSTS on plain HTTP is meaningless and could cause issues in development. Only set it when the request is known to be behind TLS.
