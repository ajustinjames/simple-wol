# Issue #12 — Set up Dependabot and GitHub security tools

**Status:** refined
**Branch:** —
**Issue:** https://github.com/ajustinjames/simple-wol/issues/12

## Spec

### Problem

The repository has no automated dependency monitoring, code scanning, or security tooling configured.

### Implementation

**Add `.github/dependabot.yml`** for automated dependency update PRs:
```yaml
version: 2
updates:
  - package-ecosystem: "gomod"
    directory: "/"
    schedule:
      interval: "weekly"
  - package-ecosystem: "docker"
    directory: "/"
    schedule:
      interval: "weekly"
```

**Add `.github/workflows/codeql.yml`** for static analysis:
- Trigger on push to `main` and PRs targeting `main`
- Use `github/codeql-action/init` and `github/codeql-action/analyze` with language `go`
- Run on `ubuntu-latest`

**Add `SECURITY.md`** with a vulnerability disclosure policy:
- Contact method (GitHub issues or email)
- Expected response time
- Scope of the policy

**Branch protection and secret scanning** are repository settings, not code changes. Document these as manual steps in the PR description:
- Enable branch protection on `main`: require PR reviews, require status checks
- Enable secret scanning and push protection in repository settings

**Files:** `.github/dependabot.yml` (new), `.github/workflows/codeql.yml` (new), `SECURITY.md` (new)

### Test Cases

- [x] Dependabot config is valid YAML and recognized by GitHub
- [x] CodeQL workflow runs successfully on Go code
- [x] SECURITY.md is present and accessible

## Decisions

1. **Weekly schedule for both ecosystems** — Daily is too noisy for a small project. Weekly balances freshness with PR noise.

2. **CodeQL over third-party scanners** — CodeQL is free, first-party, and requires no external accounts. It covers the Go-specific vulnerability patterns relevant to this project.

3. **Branch protection documented rather than automated** — These are repository settings that can't be configured via committed files. They require admin access to the GitHub repo settings UI or the GitHub API.
