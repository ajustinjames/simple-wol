# Issue #8 — Fix rate limiter memory leak and add eviction

**Status:** refined
**Branch:** —
**Issue:** https://github.com/ajustinjames/simple-wol/issues/8

## Spec

### Problem

The `RateLimiter` in `auth.go:96-158` stores login attempts in `map[string]*loginAttempt` with no eviction. If many distinct IPs trigger failed logins, the map grows without bound. Entries are only removed when:
- The IP successfully logs in (`Reset`)
- The IP tries again after the window/lockout expires (`Allow`)

IPs that fail and never return stay in the map forever.

### Implementation

**Add a `StartCleanup` method on `RateLimiter`** that launches a goroutine with a ticker (every 10 minutes). On each tick, iterate the map under the lock and delete entries where:
- `lockedAt` is non-zero and `time.Since(lockedAt) > lockout`
- `lockedAt` is zero and `time.Since(firstFail) > window`

**Call `app.limiter.StartCleanup()` in `main()`** after creating the limiter (around main.go:57).

**Files:** `auth.go`, `main.go`

### Test Cases

- [x] Entries older than the window are evicted by cleanup
- [x] Locked entries older than the lockout duration are evicted
- [x] Active entries within the window are NOT evicted
- [x] Locked entries within the lockout duration are NOT evicted
- [x] Map is empty after all entries expire and cleanup runs

## Decisions

1. **No maximum IP count cap** — For a single-user home tool, the realistic number of distinct attacking IPs is small. The cleanup goroutine is sufficient. A cap would add complexity (LRU eviction, deciding what to evict) for a scenario that's unlikely.

2. **10-minute cleanup interval** — Matches the default window (10 min) and lockout (15 min) durations. Frequent enough to prevent meaningful buildup.

3. **No persistence of rate limit state across restarts** — The audit mentioned this, but for a home tool the risk is minimal. An attacker who can restart the server already has more access than rate limiting protects against. Adding database persistence would significantly complicate the rate limiter.
